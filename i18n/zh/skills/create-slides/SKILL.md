# /create-slides

> 在 `wiki/outputs/` 下编写或更新 Markdown 幻灯片文件，遵循 `docs/slides-guide.zh.md` 中的统一规则集。
> 默认引擎：**Obsidian + 插件 [`obsidian-advanced-slides`](https://github.com/MSzturc/obsidian-advanced-slides)**（底层为 Reveal.js）。

## 调用方式

手动命令：`/create-slides <output-path> [--from <source>] [--type <slide-type>]`

## 输入参数

- `<output-path>`（必需）：需要创建/更新的幻灯片文件路径，例如 `wiki/outputs/my-talk-slides.md`
- `--from <source>`（可选）：构建幻灯片的内容来源。可以是：
  - wiki 页面的 slug 或路径（`papers/`、`topics/`、`Summary/`、`experiments/`、`claims/`、`ideas/`）
  - 现有原始笔记文件的路径
  - 用逗号分隔的 slug 列表（综合多个来源时）
  - 用户将在请求中直接描述内容时省略
- `--type <slide-type>`（可选）：演示类型，用于选择适当的模式。支持的值：
  - `paper-review` — 多篇论文的相关工作/文献综述（默认使用 7.1 节的 3 层流程）
  - `ideas` — 想法/方法提案（7.2 节的理论→原理模式）
  - `results` — 实验结果报告（7 节的结果/数据幻灯片）
  - `talk` — 综合演讲/研究总结（结合多种模式）
  - 省略 → 从内容自动检测或询问用户

## 输出

- `<output-path>` 处的 Markdown 幻灯片文件包含：
  - 用于 Reveal.js 配置的 YAML 前置元数据 + 用于左对齐的 `<style>` 块
  - 标题幻灯片、具有正确层级结构的内容幻灯片、结束幻灯片
  - 按照指南规则编写的公式、表格、图片、参考文献
- 如果输出位于 `wiki/outputs/` 下，则更新 `wiki/index.md`
- 追加到 `wiki/log.md`：`## [{date}] create-slides | <简短描述>`

## Wiki 交互

### 读取
- 必需：`docs/slides-guide.zh.md` — 唯一规则来源；在编写任何幻灯片之前必须打开
- 按需：用户通过 `--from` 指定的 wiki 页面
- 可选：如果输出是 `outputs/` 之外的 wiki 页面，则读取 `docs/runtime-page-templates.vi.md`

### 写入
- `<output-path>`（新建或更新的幻灯片文件）
- `wiki/index.md`（如果输出属于列出的类别）
- `wiki/log.md`（仅追加）

## 步骤

### 第 1 步：加载规则与确定范围

1. **完整阅读 `docs/slides-guide.zh.md`** — 这是所有格式、风格、来源和幻灯片模式规则的唯一来源。在当前会话中未重新阅读指南之前不得编写幻灯片。
2. 确定：
   - 幻灯片类型（`--type` 或根据请求推断）
   - 主要语言（VI / EN）— 整个文件保持一致
   - 估计幻灯片数量及主要流程
3. 如果主题/范围信息不足，请**先询问用户再起草**；不得捏造内容。

### 第 2 步：收集有来源的内容

1. 对于 `--from` 中的每个来源，阅读页面以提取：
   - 标准标题、发表地点/年份、重要性评分
   - 核心方法、关键公式、定量结果
   - 用于幻灯片底部引用的 slug
2. 按照 7.3 节的规定对每个要放在幻灯片上的声明进行分类：
   - **有来源的事实** — 必须包含 slug/引用/链接
   - **实验观察** — 必须包含实验 slug + 种子 + 度量指标
   - **假设** — 必须用明确的措辞标记
3. **不得捏造数字、引用或声明**。如果来源缺乏必要信息，以假设形式编写或省略。

### 第 3 步：构建幻灯片框架

按照 `docs/slides-guide.zh.md` 中的规定应用与 `--type` 匹配的模式：

- **`paper-review`** → 7.1 节的 3 层流程：主题概述幻灯片 → 每篇论文一页幻灯片（标题/方法/结果/引用）→ 总结与差距幻灯片。
- **`ideas`** → 7.2 节模式：概览表格幻灯片 → 每个想法一页幻灯片（理论→原理）。
- **`results`** → 7 节的"结果/数据幻灯片"，使用两栏度量指标表格 + 图片布局。
- **`talk`** → 组合：标题 → 动机 → 相关工作（paper-review）→ 贡献（ideas）→ 结果 → 结论。

始终包含：
1. 标题幻灯片（H1 + 副标题 + 作者 · 日期）
2. 目录幻灯片（如有多个部分）
3. 包含内容幻灯片（H2）的部分（H1）
4. 结束/问答/整体参考文献幻灯片

### 第 4 步：按照指南编写幻灯片

每张幻灯片必须通过以下规则（必要时与指南交叉核对）：

1. **前置元数据**（第 1 节）：YAML + `<style>` 块；`center: false`、`text-align: left`、按指南设置字体大小。
2. **层级结构**（第 2 节）：H1 用于打开部分，H2 用于内容幻灯片，H3 用于子标题，`---` 用于幻灯片分隔。
3. **两栏**（第 3 节）：`<split even gap="2">`，文本放在 `<div>` 内，图片放在 `<div>` 外。
4. **表格**（第 4 节）：对齐 `:---:` / `---:`，重要行加粗，次要表格使用 `<small>`。
5. **公式**（第 5 节）：
   - 仅使用 `$$...$$`（块）和 `$...$`（行内）；禁止 LaTeX 环境
   - 指数用 `{}` 包裹：`x^{2}`、`\nabla^{2}`、`10^{-3}`
   - 多字符下标用 `{}` 包裹：`\mathcal{L}_{\mathrm{total}}`
   - 单字符下标（`\mu_1`、`K_2`）保持不变
   - 数学文本使用 `\mathrm{}`；范数使用 `\|...\|`；乘法使用 `\cdot`
   - **独立块公式在预览时必须居中**：每个公式用 `<div style="text-align: center;">` 包裹，或添加全局 CSS 规则
6. **图片**（第 6 节）：Obsidian 语法 `![[path|widthxheight]]`；始终指定尺寸。
7. **写作风格**（第 7 节）：
   - 专业、中立、不过度简略、阅读流畅
   - 避免广告语气、命令式、感叹式、非标准缩写
   - 加粗用于名称/度量指标/关键词；`—`、`→`、`·` 在适当上下文中使用
8. **来源**（第 7.3 节）：
   - 每个事实必须有权威来源（同行评审/官方文档/有日志的实验）
   - 不得引用博客/推文/未经验证的 AI 生成内容
   - 每个数字附带度量指标/基线/数据集/slug
   - 假设用"可能"、"预期"、"假设"等措辞明确标记
9. **引用**（第 8 节）：使用外部来源时在幻灯片底部使用 `<small>Ref: slug-a · slug-b</small>`。

### 第 5 步：按照第 11 节检查清单进行自查

在报告完成之前，逐项检查第 11 节检查清单中的每一条——分为两组：

**格式与风格：**
- [ ] 完整的前置元数据 + `<style>` 左对齐 + `center: false`
- [ ] 仅部分开头使用 H1；`---` 前后有空行
- [ ] 图片有 `|widthxheight`；表格在需要时使用对齐标记
- [ ] 公式使用 `$$...$$` / `$...$`；指数用 `^{}`；多字符下标用 `_{}`
- [ ] 独立块公式居中（方法 1 或方法 2）
- [ ] 通过 `<small>` 在幻灯片底部标注来源
- [ ] 专业、中立、流畅的写作风格
- [ ] 使用一致的语言；内容不溢出幻灯片

**来源与真实性：**
- [ ] 每个事实都有来源；不将博客/推文/猜测作为事实
- [ ] 假设已明确标记
- [ ] 实验观察包含实验 slug + 种子 + 度量指标
- [ ] 每个数字都有完整的上下文（度量指标/基线/数据集/来源）
- [ ] 事实和假设不在同一句子中混合
- [ ] 幻灯片底部引用格式正确
- [ ] 文献声明附有置信度（如有）

如果任何项目未通过 → 在报告前修复幻灯片。

### 第 6 步：更新导航与日志

1. 如果输出位于 `wiki/outputs/` 下，在 `wiki/index.md` 的 `Outputs` 类别下添加条目。有关确切格式，请参阅 `docs/runtime-support-files.vi.md`。
2. 追加到 `wiki/log.md`：
   ```markdown
   ## [{YYYY-MM-DD}] create-slides | <简短内容描述>
   ```
3. **不要触碰 `graph/`**。

### 第 7 步：报告

- 列出：文件路径、幻灯片数量、幻灯片类型、使用的来源
- 1-2 行主要内容摘要
- 提醒用户使用 `obsidian-advanced-slides` 打开预览，并目视检查公式居中和栏布局

## 约束条件

- **`docs/slides-guide.zh.md` 是唯一的规则来源** — 每个格式/风格/来源的决定都必须查阅此文件；不得自创新规则。
- **不得捏造内容**：如果来源缺失，则不编写；如果声明缺乏依据，按照 7.3 节将其标记为假设。
- **禁止 LaTeX 环境**（`\begin{equation}`、`\begin{align}`）— Reveal.js 上的 MathJax 对它们支持不稳定。
- **指数总是用 `{}` 包裹**：`x^{2}`，而不是 `x^2`；多字符下标用 `{}` 包裹；单字符下标保持不变。
- **独立块公式在预览时必须居中**（按指南第 5 节）。
- **整个文件使用一种主要语言**；标准化的技术术语可以保留英文。
- **来源是强制性的**：在报告完成前必须满足第 11 节"来源与真实性"检查清单。
- **不要编辑 `graph/`**；更新 `index.md` 并追加到 `log.md`。
- **如果现有幻灯片文件不是由本技能生成的，在覆盖前必须获得用户确认**。
